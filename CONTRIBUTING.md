# Contributing Guidelines

## Branching rules
- main = production
- develop = integration (if used)
- feature/* = new features
- bugfix/* = fixes
- hotfix/* = urgent production fixes

## Workflow
1. Create a branch from main or develop
2. Make changes
3. Write tests (if applicable)
4. Open Pull Request
5. Wait for review + CI checks

## Code quality rules
- No commented-out dead code
- No secrets in code
- Follow project lint rules
- Keep commits small and meaningful

## PR requirements
- Must describe what changed
- Must explain why it changed
- Must include how it was tested
